## 🤖 一、自主运行机制

### 1.1 核心原则

```
✅ 自主决策 - 根据优先级自主决定做什么
✅ 自主执行 - 自动执行任务，无需人工启动
✅ 自主修复 - 遇到问题自动尝试修复
✅ 自主汇报 - 定期主动汇报进度
✅ 自主迭代 - 持续改进，永不满足
```

### 1.2 执行循环

```
while true; do
  1. 检查系统状态 (应用/数据库/服务)
  2. 从任务队列获取最高优先级任务
  3. 执行任务
  4. 验证结果
     - 成功 → 记录 → 汇报 → 继续
     - 失败 → 自动重试 (3 次)
       - 仍失败 → 记录问题 → 调整方案 → 重试
  5. 更新任务队列
  6. 等待 5 分钟 (避免过度执行)
  7. 回到步骤 1
done
```

## 🔧 三、自主执行器

### 3.1 主执行脚本

```bash
#!/bin/bash
# autonomous_executor.sh - 完全自主执行器

TASK_QUEUE="/workspace/wechat-acquisition/.task_queue"
LOG_FILE="/tmp/autonomous_executor.log"
REPORT_FILE="/tmp/autonomous_report.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" | tee -a "$LOG_FILE"
}

get_next_task() {
    # 获取最高优先级待执行任务
    grep -E "^(P0|P1|P2|P3)\|" "$TASK_QUEUE" | \
    grep "|TODO|" | \
    sort -t'|' -k1,1 | \
    head -1
}

execute_task() {
    local task="$1"
    local task_id=$(echo "$task" | cut -d'|' -f2)
    local task_desc=$(echo "$task" | cut -d'|' -f3)
    
    log "▶️  执行任务：$task_id - $task_desc"
    
    case "$task_id" in
        T001) check_system_status ;;
        T002) improve_data_import ;;
        T101) enhance_customer_tags ;;
        # ... 更多任务
        *) log "❌ 未知任务：$task_id" ;;
    esac
}

# 主循环
log "╔══════════════════════════════════════════════════════════╗"
log "║   完全自主执行器已启动                                   ║"
log "╚══════════════════════════════════════════════════════════╝"

while true; do
    task=$(get_next_task)
    
    if [ -n "$task" ]; then
        execute_task "$task"
    else
        log "✅ 暂无待执行任务，等待 5 分钟..."
    fi
    
    sleep 300  # 5 分钟
done
```

### 3.2 系统状态检查

```bash
check_system_status() {
    log "检查系统状态..."
    
    # 检查应用
    if ! curl -s http://localhost:8080/api/health | grep -q '"status":"UP"'; then
        log "❌ 应用未运行，尝试重启..."
        restart_application
        add_task "P0" "T001" "应用重启验证"
        return 1
    fi
    
    # 检查数据库
    if ! docker ps | grep -q mysql; then
        log "❌ MySQL 未运行，尝试启动..."
        start_mysql
        add_task "P0" "T001" "数据库连接验证"
        return 1
    fi
    
    log "✅ 系统状态正常"
    return 0
}
```

### 3.3 自动重试机制

```bash
execute_with_retry() {
    local task="$1"
    local max_retries=3
    local retry_count=0
    
    while [ $retry_count -lt $max_retries ]; do
        if execute_task "$task"; then
            mark_task_done "$task"
            return 0
        else
            retry_count=$((retry_count + 1))
            log "⚠️  执行失败，${retry_count}/${max_retries} 次重试"
            
            if [ $retry_count -lt $max_retries ]; then
                local wait_time=$((2 ** retry_count * 60))
                log "等待 ${wait_time}秒后重试..."
                sleep $wait_time
            fi
        fi
    done
    
    mark_task_failed "$task"
    log "❌ 达到最大重试次数，任务失败"
    send_alert "任务失败：$task"
    return 1
}
```
